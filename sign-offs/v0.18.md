### v0.18 Specific Proof Criteria

- [x] **Join lowering**: `LogicalPlan::Join` lowers to `PlanNode::Join` with
  AND-folded equijoin conditions; cross joins use constant-true literal.
  Tests: `lower_inner_join_produces_join_node`,
  `lower_cross_join_produces_join_with_true_condition`,
  `lower_aggregate_over_join_produces_correct_structure`.

- [x] **`rockstream sql` subcommand**: parses SQL via DataFusion with built-in
  demo schema (`orders`, `products`, `events`), lowers to PlanNode IR, and
  prints `EXPLAIN INCREMENTAL` output. Test: `cli_parses_sql_subcommand`.

- [x] **SQL Alpha correctness soak**: all Phase 1 operators (filter, project,
  aggregate×5, join, union) pass lowering + explain + law-annotation checks
  in `sql_alpha_soak_all_phase1_operators`.

- [x] **DDL parse**: `CREATE VIEW` and `CREATE MATERIALIZED VIEW` parse without
  error. Tests: `ddl_create_view_parses`, `ddl_create_materialized_view_parses`,
  `ddl_multiple_statements_parse`.

- [x] **One-hour fuzzer — no divergence**: deterministic fuzzer (`SEED =
  0x5EED_1850_ADEF_0018`, 256 iterations) generates random Phase 1 PlanNode
  trees and verifies no panic, every aggregate annotated with law or reason,
  explain output consistent, and re-running same plan produces identical
  annotations. Test: `sql_alpha_fuzzer_no_divergence`.

- [x] **Fuzzer seed stability**: same seed always produces identical plan
  sequence (RNG reproducibility). Test: `sql_alpha_fuzzer_seed_stability`.

- [x] **Catalog join round-trip**: `PlanNode::Join` encodes and decodes through
  catalog codec without data loss. Test: `proof_v018_join_plan_round_trips_through_codec`.

- [x] **Catalog DiffCtx no-divergence**: in-memory and catalog-loaded plans
  produce identical DiffCtx operator annotations.
  Test: `proof_v018_difctx_no_divergence_across_catalog_load`.

- [x] **All Phase 1 catalog proofs**: every operator type round-trips through
  catalog codec and DiffCtx annotations are stable.
  Test: `proof_v018_all_phase1_operators_annotated_consistently`.

- [x] **Explain join integration**: explain output for join plans includes
  Source and Join rows; aggregate over join has law annotation.
  Tests: `explain_join_plan_contains_join_row`,
  `explain_filter_join_aggregate_has_all_rows_and_law`.
