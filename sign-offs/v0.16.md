### v0.16 Specific Proof Criteria

- [x] A workload can be created with `FRESHNESS_SLO`, `MEMORY_LIMIT`, and `PRIORITY`
  - Proof: `proof_create_workload_and_assign_to_view` in `rockstream-catalog/src/lib.rs`
- [x] A view can be created referencing a workload (`WITH WORKLOAD = name`)
  - Proof: `proof_create_workload_and_assign_to_view`
- [x] Creating a view with a non-existent workload returns RS-1005
  - Proof: `proof_register_view_with_unknown_workload_fails`
- [x] Namespace-level default workload is applied to views without explicit assignment
  - Proof: `proof_namespace_default_workload`
- [x] `PAUSE MATERIALIZED VIEW` transitions state to `Paused` with audit event
  - Proof: `proof_pause_and_resume_view_with_audit_events`
- [x] Double-pause returns RS-1007 (`ViewAlreadyPaused`)
  - Proof: `proof_pause_and_resume_view_with_audit_events`
- [x] `RESUME MATERIALIZED VIEW` transitions state back to `Running` with audit event
  - Proof: `proof_pause_and_resume_view_with_audit_events`
- [x] Resuming a running view returns RS-1008 (`ViewNotPaused`)
  - Proof: `proof_pause_and_resume_view_with_audit_events`
- [x] `SHOW VIEW STATUS FOR NAMESPACE` reports current state and SLO
  - Proof: `proof_show_view_status_reports_state_and_slo`
- [x] `SHOW BACKFILL STATUS FOR MATERIALIZED VIEW` returns backfill progress
  - Proof: `proof_show_backfill_status_for_materialized_view`
- [x] Dropping a workload with assigned views is rejected
  - Proof: `proof_drop_workload_in_use_is_rejected`
- [x] All new error codes have descriptions and next-steps in the registry
  - Proof: `all_codes_have_descriptions` in `rockstream-types/src/error_code.rs`
- [x] View dependency metadata (`depends_on`) is stored per entry
  - Proof: `proof_create_workload_and_assign_to_view`
- [x] `WorkloadId` added to `rockstream-types::ids`
- [x] `WorkloadDef`, `FreshnessSlo`, `MemoryLimit`, `WorkloadPriority` types in `rockstream-types::workload`
- [x] `ViewState`, `ViewStatus`, `BackfillStatus` types in `rockstream-types::view_lifecycle`
- [x] `WorkloadStore` in `rockstream-catalog::workload_store`
- [x] `CatalogStore` extended with all v0.16 DDL and lifecycle methods
- [x] New error codes RS-1005, RS-1006, RS-1007, RS-1008 registered in error_code module
