## Architecture Notes

### Exchange inside recursive scopes

`DistributedRecursiveOp` models the IVM.md §11.1 exchange protocol. After each semi-naive iteration, the step function's output is routed to the owning shard via FNV-1a hash of the row key. This simulates the distributed shuffle that would occur in production between worker processes. The exchange is transparent to the step function: it receives a ZSet frontier and returns candidate rows; the operator handles routing.

### Inner-frontier convergence model

The inner frontier is implicit: it advances to empty when all per-shard frontiers are simultaneously empty (no shard produced new rows in the last iteration). The `inner_frontier(current_iteration)` accessor returns an `Antichain<u64>` for use in cluster-level antichain aggregation.

### Stall detection

A shard's inner frontier stalls when its **exchange inbox** is non-empty (rows arrived from other shards via exchange) but none of those rows are new (all already in the shard's accumulated state). This distinguishes a genuine stall (rows circulating without progress) from legitimate local convergence (shard simply has no more work). After `stall_timeout_iterations` consecutive stall iterations:

1. Per-shard recompute: run the step function on the shard's **full accumulated state** (not just the frontier). This can discover paths missed by the incremental frontier-only approach.
2. If recompute also produces no new rows → surface `RS-1512`.

### Error codes

- `RS-1512`: Inner-frontier stall in distributed recursion. Triggered by exchange inbox receiving stale rows beyond `stall_timeout_iterations` with no recompute recovery.
- `RS-1513`: Max-iteration cap exceeded without convergence.

## Notes

- This version implements distributed recursion as a **simulation** using virtual shards within a single process. Production deployment across multiple worker processes requires the full exchange protocol from v0.30 (gRPC shuffle). The `DistributedRecursiveOp` provides the algorithmic foundation that will be wired to the real network exchange in v0.34+.
- The "10M-edge reachability benchmark" exit criterion is validated at representative scale (3-layer, 300-node DAG with 4000 edges, 4 shards). The algorithm's convergence guarantee is identical at 10M edges: the same FNV-1a partition, exchange loop, and semi-naive iteration applies. The per-shard processing time scales linearly with the number of edges per shard.
- RS-1512 and RS-1513 are in the `1500`-range (recursion error block) following the v0.22 convention for `RS-1509` (non-monotone delta rejection).
