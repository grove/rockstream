## Architecture Notes

### Failure detection

`WorkerHealthMonitor` accepts a deterministic millisecond clock, enabling exact SLO proof tests without any real wall-clock dependency. `tick(now_ms)` returns failed workers in O(n) by scanning all registered heartbeat timestamps.

### Recovery tracking

`RecoveryDriver` records recovery events and checks SLO via `status(now_ms)`. The driver is intentionally decoupled from `ShardScheduler` ‚Äî the caller invokes `on_worker_dead` and passes the resulting reassignments to `record_recovery`. This keeps the recovery driver pure and testable.

### RECOVERING_SLOW (RS-1603)

`RecoveryStatus::RecoveringSlow` fires when any active recovery has been running longer than `slow_threshold_ms` (60 000 ms by default). The `elapsed_ms` field allows operators to observe how far beyond SLO the recovery has run.

### Control-plane self-fencing

`ControlPlaneFence` implements DESIGN.md ¬ß11.6: a worker that loses contact with the control plane for more than `fence_timeout_ms` (30 000 ms) transitions to `Fenced` and must stop committing. This prevents a partitioned worker from writing to a shard whose lease has been revoked and reassigned.

### Thundering herd prevention

`ThrottledLeaseGranter` implements DESIGN.md ¬ß11.8 with a sliding-window rate limit. The window rolls forward by whole multiples of `window_ms` when `try_grant` is called in a new window, preserving the budget correctly across multi-window gaps.

### Error code cleanup

v0.29 `shard.rs` was using RS-1601/1602/1603 as inline error strings ‚Äî the same numbers that v0.34 registered for checkpoint errors. This release fixes the collision by:
- Moving shard lease errors to RS-1701/1702/1703 (in `shard.rs` inline strings and now formally registered in `error_code.rs`)
- RS-1601/1602 remain for checkpoint alignment buffer overflow and RECOVERING state (v0.34)
- RS-1603 is now RECOVERING_SLOW (v0.35)

## Notes

- `WorkerHealthMonitor`, `RecoveryDriver`, `ControlPlaneFence`, and `ThrottledLeaseGranter` are all pure synchronous data structures with no async dependencies. Production wiring to the real gRPC control plane and `SimRuntime` clock will be done in v0.36.
- The "32-worker simultaneous restart" SLO is proved at the detection level (no false positives at 1 s tick). The full end-to-end recovery timeline (detection + reassignment + first commit) is bounded by the sum of individual SLOs: 5 s + 30 s + 60 s = 95 s worst case.
