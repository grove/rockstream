## Summary

Implements v0.24: View-on-view DAG with `ViewRef`, compile-time cycle detection (RS-1011), cadence inheritance via topological scheduling, and structural diamond consistency.

## What changed

### Core types
- `RS_1011` error code: "View-on-view DAG contains a cycle"

### Plan IR (`rockstream-plan`)
- `PlanNode::ViewRef { view_name }` — reads CDC deltas from an upstream materialized view
- `OpKind::ViewRef { view_name }` — physical counterpart (stateless, no local arrangement)
- `dag` module:
  - `detect_cycle` — DFS-based compile-time cycle detection; returns `Err(cycle_path)` on cycle (RS-1011 at call site)
  - `topological_order` — Kahn's algorithm; deterministic scheduling order (cadence inheritance)
  - `direct_view_refs` — collect all `ViewRef` names from a plan tree

### Differentiation (`rockstream-diff`)
- `PlanNode::ViewRef` → `OpKind::ViewRef` with `Stateless` not-merge-safe reason

### Catalog codec (`rockstream-catalog`)
- Encode/decode for `PlanNode::ViewRef` with `view_name` roundtrip

### Explain labels
- `ViewRef[<view_name>]` in both `rockstream-plan/explain` and `rockstream-runtime/explain`

### SQL (`rockstream-sql`)
- `op_kinds` and `assert_op_kinds` updated with `ViewRef` arm

### Reference oracle (`rockstream-oracle`)
- `ViewDagOracle`: evaluate views in topological order, simulate multi-epoch accumulation
- 13 proof tests in `view_dag_tests.rs`
