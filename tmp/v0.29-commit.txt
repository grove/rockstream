v0.29.0: Shard leasing and scheduling

Implements the v0.29 roadmap entry: shard lease management with
monotone fencing tokens, writer fencing, distributed shard scheduling,
and TCP wire protocol for RequestShard/FenceWrite.

## What is new

### rockstream-types: ShardLease and ShardRevokeReason

- New module lease.rs: ShardLease (shard_id, worker_id, lease_token,
  acquired_at_ms) and ShardRevokeReason (WorkerDead / ControlPlanePreempt).
- ControlMessage extended: ShardAssigned, ShardRevoked, FenceAck variants.
- WorkerMessage extended: RequestShard, FenceWrite variants.

### rockstream-control: ShardManager

- Thread-safe in-memory shard lease store (Arc + RwLock).
- acquire: issues a fresh monotone LeaseToken; same worker can reacquire
  (token increases); different worker gets RS-1601 AlreadyLeased.
- force_acquire: evicts any current holder (used by control plane for
  rebalancing); returns evicted WorkerId.
- release / release_worker: clean removal of one lease or all leases for
  a dead worker.
- is_valid_writer: the write fence — checks that the provided token is
  the current active token for the shard.
- Tokens are globally monotone across all shards and all workers:
  every new acquire increments a single counter.
- 13 unit tests including the required two_writer_fence_test.

### rockstream-control: ShardScheduler

- Combines TopologyCatalog + ShardManager + PlacementAlgorithm.
- assign_initial_shards: assigns a list of shards to healthy workers
  by descending capacity_headroom; skips shards already held by a
  healthy worker.
- on_worker_dead: releases all leases for the dead worker, then
  reassigns each freed shard to the next best healthy worker with a
  fresh fencing token.
- 5 unit tests including worker_death_causes_reassignment and
  worker_death_with_no_remaining_workers_frees_shards.

### rockstream-control: ControlService extended

- New with_shard_manager builder to inject a shared ShardManager.
- handle_connection now tracks connected_worker_id.
- RequestShard: calls shard_manager.acquire; replies ShardAssigned on
  success; closes connection on RS-1601 (different worker holds lease).
- FenceWrite: calls is_valid_writer; replies FenceAck{valid}.
- Deregister: calls release_worker, emits ShardRevoked for each freed
  shard, then TopologyChanged.
- Disconnect (TCP drop without Deregister): release_worker cleans up
  all leases; topology catalog entry is preserved.
- 4 new integration tests including shard_leases_released_on_worker_disconnect.
