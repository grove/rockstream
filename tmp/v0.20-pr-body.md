# v0.20.0: Time windows, watermarks, and MaxRegister/v1

## Summary

Implements tumbling time-window (TUMBLE) operators with event-time watermarks. Escape hatch applied: TUMBLE only implemented; HOP/SESSION deferred to v0.21.

## Changes

### New: `LateDataPolicy` (rockstream-plan)

Three policies for rows that arrive after a window has closed:
- `Drop` â€” silently discard; closed window is unchanged
- `Update` â€” retract previous window output and re-emit with late row included
- `RouteToSink { sink_name }` â€” route late row to a named side-channel sink

### New: `PlanNode::TumbleWindow` (rockstream-plan)

Logical plan node with fields: `time_col`, `window_size_ms`, `late_data_policy`.

### New: `OpKind::TumbleWindow` (rockstream-plan)

Physical operator kind. `DiffCtx` assigns `merge_law = MaxRegister/v1` (semilattice, idempotent watermark tracking). No `not_merge_safe_reason` needed.

### New: `TumbleOp` (rockstream-ops)

IVM operator with:
- Watermark advances via `MaxRegister/v1` semantics (`max(current, new)` â€” never retreats)
- Windows close exactly once when watermark passes their end
- `emitted` state retained indefinitely (TTL never removes visible state)
- Idempotent: duplicate watermark values produce no output

### New: `TumbleOracle` (rockstream-oracle)

Batch reference implementation for correctness verification.

### Codec

Full encode/decode for `TumbleWindow` and `LateDataPolicy` (all three variants) in `rockstream-catalog`.

### EXPLAIN

`TumbleWindow[{size}ms,{policy:?}]` reported in plan, runtime, and SQL explain paths.
