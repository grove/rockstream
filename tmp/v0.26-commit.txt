v0.26.0: IVM correctness freeze with law equivalence

Implements the full v0.26 scope: TPC-H 22/22, Nexmark Q1-Q8, random query
fuzzer, and law-equivalence corpus.

## New oracle modules

- `tpch_oracle`: Schema helpers and batch reference evaluators for all 22
  TPC-H query shapes (Q1-Q22). Covers filter, join, aggregate, MIN/MAX,
  EXISTS subquery, HAVING, distinct count, profit, promotion, and top-k.

- `nexmark_oracle`: Nexmark event types (Bid, Auction, Person) and batch
  reference evaluators for Q1-Q8: currency conversion, item filtering,
  local suggestion, category avg, hot items, seller avg, max price, and
  new user monitoring.

- `fuzzer_oracle`: `FuzzedPipeline` with 6 `PipelineShape` variants
  (Filter, Project, Aggregate, FilterAggregate, Join, JoinAggregate);
  `FuzzerOracle` tracking divergence count. Designed for extended soak
  runs via `PROPTEST_CASES`.

- `law_equiv_oracle`: Per-law equivalence checkers for all 6 registered
  laws (WeightAdd, SumCount, MaxRegister, MinRegister, HyperLogLog,
  BloomUnion); `EquivalenceCorpus` running all 6 in one call.

## New test files

- `tpch_tests.rs`: 22 proof tests (one per TPC-H query shape Q1-Q22).
- `nexmark_tests.rs`: 8 proof tests (one per Nexmark query Q1-Q8).
- `fuzzer_tests.rs`: 15 tests: 8 deterministic + 7 proptest fuzz cases.
- `law_equiv_tests.rs`: 21 tests: 18 deterministic + 6 proptest
  equivalence checks (one per law).

## Exit criteria met

- 22/22 TPC-H queries match batch reference (all query shapes Q1-Q22).
- Nexmark Q1-Q8 IVM output matches batch reference.
- Random query fuzzer: 6 pipeline shapes x proptest cases, zero divergences.
- Law-equivalence corpus: all 6 registered laws report zero divergence.
- All tests pass, none skipped.
- `cargo clippy --workspace --all-targets -- -D warnings` clean.
- `cargo fmt --all` applied.
