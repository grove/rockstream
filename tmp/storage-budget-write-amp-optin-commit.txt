Make storage_budget write-amplification probe explicit opt-in

Gate mode was still perceived as hanging when Azure was configured because
run_write_amplification_probe performs many sequential put() calls and can
take several minutes.

Change behavior:
- Keep STORAGE_BUDGET_RUN_GATES=1 for gate probes (WAL + latency)
- Run write-amplification only when STORAGE_BUDGET_RUN_WRITE_AMP=1 is set
- Print a clear GATE SKIP message when write-amplification is disabled

This keeps normal gate runs fast and predictable while preserving a manual
path for full sign-off evidence collection.
