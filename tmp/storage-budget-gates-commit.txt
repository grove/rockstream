Fix storage_budget bench hang: gate probes now guarded by env var

Gate probes (WAL cache, latency, write-amp) are long-running diagnostics
that hang when invoked inside the Criterion harness. Wrapping them with
STORAGE_BUDGET_RUN_GATES=1 env var guard makes `cargo bench` fast (~30s)
by default, and only runs probes when explicitly requested.

Usage:
  cargo bench -p rockstream-ops --bench storage_budget    # fast, benches only
  STORAGE_BUDGET_RUN_GATES=1 \
    cargo bench -p rockstream-ops --bench storage_budget  # full gates (requires Azure env)
