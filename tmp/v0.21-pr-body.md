# v0.21.0: Top-K and approximate sketches

Implements the v0.21 roadmap milestone: Top-K detection with K+epsilon state,
delete refill path, delta swaps, partitioned Top-K, and the HyperLogLog/v1
semilattice merge law for planner NDV estimation.

## What's in this release

### HyperLogLog/v1 merge law

A new `HyperLogLog/v1` semilattice law (ID `0x0005`) is registered in the
global law registry alongside the four existing laws. The law merges two
64-register HLL sketches by taking the per-register maximum — making it
associative, commutative, and idempotent. It is used by the planner cost model
to estimate the number of distinct values (NDV) in a column.

- Wire format: 64 bytes (one byte per register)
- Identity: all-zero sketch
- Hash: FNV-64a with splitmix64 finalizer for uniform bit distribution
- NDV estimator: HyperLogLog formula with small-range linear-counting correction

The escape hatch was not triggered: HLL accuracy was sufficient for
cost-model correctness.

### Top-K operator (`TopKOp`)

A new physical operator that maintains the top-`k` rows per partition ranked
by a score column, emitting incremental delta swaps when the ranked set changes.

Key design decisions:
- **K+epsilon buffer** (epsilon=k, total buffer=2k rows): backs the delete-refill
  path without a full state rescan
- **Delete-refill path**: deleting a row from the current top-K brings in the
  next-best row from the buffer
- **Delta swaps**: a new row outranking the current k-th row emits a retract+insert
  pair in a single output ZSet
- **Partitioned Top-K**: independent top-K state per partition key, with a
  configurable `PartitionFn`
- **Emitted cache**: retains `(key, value)` for rows in the current top-K so
  retractions can be emitted even after the row is removed from live state
- **Ghost tracking**: over-retracted rows (net weight < 0) are retained until
  the net weight returns to 0, ensuring correct IVM semantics

- `PlanNode::TopK { input, k, rank_col, partition_by }` added to the logical plan IR
