## Summary

Implements v0.19.0: Window functions for RockStream IVM.

All 8 window functions are implemented via two IVM strategies:
- **PartitionRecompute**: ROW_NUMBER, RANK, DENSE_RANK, NTILE, LAG, LEAD — not merge-safe; recomputes the full partition on each delta.
- **SlidingAggregate**: SlidingSum, SlidingAvg — merge-safe; reuses `SumCount/v1` as the sub-component law, reported in `EXPLAIN`.

## Changes

### Core IR and Differentiation
- `PlanNode::Window`, `WindowExpr`, `WindowFunc` (8 variants), `WindowStrategy`, `OpKind::Window` added to `rockstream-plan`
- `DiffCtx` handles `PlanNode::Window` with correct strategy/law/reason selection
- `NotMergeSafeReason::PartitionRecomputation` added to the closed enum in `rockstream-types`

### Operator Implementation
- `rockstream-ops/src/window.rs`: `WindowOp` with net-weight tracking for correct retraction-before-insert semantics, BTreeMap partition state sorted by (order_key, row_id)

### Codec and Explain
- `rockstream-catalog/src/codec.rs`: full encode/decode of `PlanNode::Window` and all `WindowFunc` variants
- `explain_op_node` / `kind_label` updated in plan and runtime explain

### Property Tests (10 total, all passing)
- 5 proptest: IVM output matches batch reference oracle for all 8 functions under randomized insert/delete/re-insert sequences
- 5 proof: SumCount law in EXPLAIN, PartitionRecompute strategy, Window codec round-trip, all variant round-trips, cost documentation
