Add storage_budget bench for Phase 2/3 budget gate

Implements crates/rockstream-ops/benches/storage_budget.rs:

- PUT/GET latency probes (200 ops, p50/p95/p99/max) with pass/fail
  verdicts against DESIGN.md §5.4 budgets (PUT < 200ms, GET < 100ms)
- WAL listing cache gate: 10k hot-path reads, asserts exactly 1 LIST
  call, prints cache hit ratio (target >99%)
- Write-amplification probe: sequential epoch writes measuring logical
  throughput; only runs when Azure env vars are set (in-memory SlateDB
  WAL-flushes-per-put makes the metric unrepresentative)
- All probes auto-skip with clear GATE SKIP messages when cloud creds
  are absent; STORAGE_BUDGET_PROBE_N env var shrinks probe size for CI

Add azure feature to rockstream-ops dev-dependencies for
object_store::azure::MicrosoftAzureBuilder.

Supports both Storage Account Key and Service Principal auth:
  AZURE_STORAGE_ACCOUNT_NAME + AZURE_STORAGE_ACCOUNT_KEY
  AZURE_STORAGE_ACCOUNT_NAME + AZURE_CLIENT_ID + AZURE_CLIENT_SECRET
                             + AZURE_TENANT_ID
