v0.19.0: Window functions (ROW_NUMBER, RANK, DENSE_RANK, LAG, LEAD, NTILE, SlidingSum, SlidingAvg)

Implements all 8 window functions via the PartitionRecompute and
SlidingAggregate IVM strategies, with full property-test coverage.

## What's new

### rockstream-types
- `NotMergeSafeReason::PartitionRecomputation` added to the closed enum
  (v0.19 version bump). `as_str()` returns `"partition_recomputation"`;
  included in `all()`.

### rockstream-plan
- `PlanNode::Window { input, window_exprs }` logical IR node.
- `WindowExpr { func, partition_by, order_by }` — per-expression descriptor.
- `WindowFunc` enum: RowNumber, Rank, DenseRank, Ntile(u64),
  Lag { offset }, Lead { offset }, SlidingSum { frame_rows },
  SlidingAvg { frame_rows }.
- `WindowStrategy` enum: PartitionRecompute | SlidingAggregate.
- `OpKind::Window { strategy }` physical operator kind.
- `explain_op_node` updated with `Window[{strategy:?}]` label.

### rockstream-diff
- `DiffCtx::diff_node` handles `PlanNode::Window`:
  - Sliding functions → `SlidingAggregate` + `SumCount/v1` merge law.
  - Ranking functions → `PartitionRecompute` + `PartitionRecomputation`
    not-merge-safe reason.

### rockstream-ops
- New `window.rs`: `WindowOp` with `PartitionKeyFn`/`OrderKeyFn`/`ValueFn`
  closures, in-memory partition BTreeMap state, net-weight tracking for
  correct retraction-before-insert handling, `process_zset()` / `Operator`
  trait impl, and `make_row_id()` stable identifier.
- `new_ranking()` for RowNumber/Rank/DenseRank/Ntile/Lag/Lead.
- `new_sliding()` for SlidingSum/SlidingAvg; reports `SumCount/v1`
  sub-law via `sub_law_id`.

### rockstream-oracle
- `window_oracle.rs`: `WindowOracle::compute()` batch reference for all 8
  window functions, used by property tests.
- `tests/window_tests.rs`: 10 tests —
  - 5 proptest: prop_row_number/rank/dense_rank/sliding_sum/sliding_avg
    match oracle (non-negative value ranges; verified IVM retraction
    correctness).
  - 5 proof: sliding-agg SumCount law in EXPLAIN, ranking PartitionRecompute
    strategy, Window plan codec round-trip, all WindowFunc variants round-trip,
    partition-recomputation cost documented.

### rockstream-catalog
- `codec.rs`: encode/decode `PlanNode::Window` with JSON `type:"Window"`;
  `encode_window_expr`, `encode_window_func`, `decode_window_expr`,
  `decode_window_func` helpers for all 8 variants.
- `lib.rs`: proof test asserts `"partition_recomputation"` is covered.

### rockstream-runtime
- `explain.rs`: `kind_label` handles `OpKind::Window { strategy }`.

### rockstream-sql
- `op_kinds` test helpers updated with `OpKind::Window` arm.

### Meta
- Workspace version bumped: 0.18.0 → 0.19.0.
- ROADMAP.md: v0.19 marked ✅ Done.
- sign-offs/v0.19.md: all checklist items checked.
