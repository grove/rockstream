v0.21.0: Top-K and approximate sketches

Implements v0.21 roadmap: Top-K operator with K+epsilon state, delete
refill path, delta swaps, and partitioned Top-K; HyperLogLog/v1 registered
as a semilattice merge law for planner NDV estimation.

## New: HyperLogLog/v1 law

- `crates/rockstream-types/src/laws/hyper_log_log.rs`
  - 64-register HLL sketch, wire format = 64 bytes
  - Merge: per-register max (semilattice: assoc, commutative, idempotent)
  - Identity: all-zero sketch
  - `hll_add` helper with FNV-64a + splitmix64 finalizer for uniform distribution
  - `hll_estimate_ndv` with small-range linear-counting correction
  - Registered as `HyperLogLog/v1` (ID 0x0005) in `LawRegistry::with_builtins()`
  - Escape hatch not triggered: HLL accuracy sufficient for cost-model correctness

## New: PlanNode::TopK / OpKind::TopK

- `PlanNode::TopK { input, k, rank_col, partition_by }` added to plan IR
- `OpKind::TopK { k, rank_col, partition_by }` added to physical graph
- DiffCtx assigns WeightAdd/v1 (abelian group) for row weight state
- Codec, explain labels, and SQL op_kind helpers updated for TopK

## New: TopKOp operator

- `crates/rockstream-ops/src/top_k.rs`
  - K+epsilon buffer (epsilon=k, buffer=2k rows per partition) for delete-refill
  - Delete-refill path: deleting a top-K row brings in the next-best from buffer
  - Delta swaps: new row outranking the k-th emits retract+insert in one output
  - Partitioned Top-K: independent state per partition key
  - `emitted_cache` retains key/value for correct retraction even after row removal
  - Ghost tracking: over-retracts kept until net weight returns to 0
  - `ScoreFn`, `PartitionFn`, `score_fn_for_col`, `no_partition_fn` helpers

## New: TopKOracle reference implementation

- `crates/rockstream-oracle/src/top_k_oracle.rs`
  - Batch reference: net weights, top-k sort, partition grouping
  - `oracle_topk_sorted` for deterministic flat comparison in tests
