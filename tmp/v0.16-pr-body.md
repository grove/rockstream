## v0.16.0: Workload DDL and View Lifecycle

Implements the v0.16 roadmap milestone.

### What's New

**Workload DDL**
- `WorkloadDef` type with `FreshnessSlo`, `MemoryLimit`, and `WorkloadPriority` (HIGH=0, DEFAULT=128, LOW=255)
- `WorkloadStore` registry scoped per namespace
- `create_workload`, `get_workload`, `drop_workload` (rejects drop if views are assigned)
- Namespace-level default workload (`set_namespace_default_workload`)
- `assign_workload_to_view` for explicit reassignment

**View Lifecycle**
- `ViewState` enum: `Running` (default), `Paused`, `BackfillingFromEpoch(u64)`
- `pause_view` / `resume_view` with audit events; double-pause → RS-1007, resume-running → RS-1008
- `show_view_status(namespace_id)` → `Vec<ViewStatus>` with state, SLO, memory limit, depends_on
- `show_backfill_status(namespace_id, view_name)` → `BackfillStatus`

**View Registration**
- `register_view_with_options` accepts `depends_on` and optional `workload_name`; validates workload exists, falls back to namespace default
- `CatalogEntry` extended with `state`, `workload_name`, `depends_on`, `backfill_started_epoch`

**New Error Codes**
| Code | Meaning |
|------|---------|
| RS-1005 | Workload not found |
| RS-1006 | Workload already exists |
| RS-1007 | View is already paused |
| RS-1008 | View is not paused |
