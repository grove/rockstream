## v0.17.0: Explain and Estimates

Implements the v0.17 roadmap milestone: `EXPLAIN INCREMENTAL` with a
finalized law-annotation contract, three explain levels (Default /
Verbose / Analyze), and backfill cost estimation with confirmation
threshold.

### What's New

**`NotMergeSafeReason` — closed enum in `rockstream-types` (v0.17 contract)**
- Moved from `rockstream-plan` to `rockstream_types::explain`
- Four variants: `ExtremumRequiresRmw`, `ClampNotALaw`, `UnknownUdafProperties`, `Stateless`
- Re-exported from `rockstream-plan` for backward compatibility — `rockstream-diff` and `rockstream-sql` are unchanged
- `LawBundle::not_merge_safe_reason()` return type updated to `Option<NotMergeSafeReason>`
- `MaxRegisterV1` and `MinRegisterV1` now return `NotMergeSafeReason::ExtremumRequiresRmw`

**`ExplainLawAnnotation` — finalized annotation contract**
- Fields: `merge_law`, `law_class`, `idempotent`, `duplicate_policy`, `compaction`, `combiner`, `partial_pushdown`, `not_merge_safe_reason`
- `merge_safe_indicator()` → `✓` (merge-safe), `⚠` (stateless), `✗` (not merge-safe)

**Three `ExplainLevel` variants (DESIGN.md §14.8)**
- `Default`: human-readable summary with `✓`/`⚠`/`✗` indicators
- `Verbose`: adds shard counts, parallelism, frontier timestamps
- `Analyze`: adds live per-operator runtime statistics (rows/s, p99 latency, state reads, RMW ratio, DLQ entries)

**Backfill cost estimation (DESIGN.md §14.9)**
- `BackfillCostEstimate` with `requires_confirmation()` and `confirmation_prompt()`
- `BACKFILL_CONFIRMATION_THRESHOLD_BYTES = 1_000_000_000` (1 GB)
- `SourceStatsEstimate`, `ConfidenceLabel` (High / Medium / Low)

**New types in `rockstream_types::explain`**

| Type | Description |
|---|---|
| `NotMergeSafeReason` | Closed enum of reasons an operator is not merge-safe |
| `ExplainLevel` | Default / Verbose / Analyze explain detail levels |
| `ExplainLawAnnotation` | Finalized law-annotation contract for `EXPLAIN` output |
| `ConfidenceLabel` | High / Medium / Low confidence tier for estimates |
| `SourceStatsEstimate` | Estimated source statistics |
| `BackfillCostEstimate` | Estimated backfill cost with confirmation threshold |
| `OperatorStats` | Live per-operator runtime statistics |
| `ShardInfo` | Shard count, parallelism, frontier epoch |
| `ExplainRow` | Single row in `EXPLAIN INCREMENTAL` output |

**New explain formatter in `rockstream_plan::explain`**
- `annotation_for(node: &OpNode) -> ExplainLawAnnotation`
- `explain_op_node(node, depth, level) -> ExplainRow`
- `format_explain_text(rows, level) -> String`
