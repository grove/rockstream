## Summary

Implements v0.22.0: **Recursion with monotone partial progress**.

Adds `PlanNode::Recursion` and a `RecursiveOp` physical operator with semi-naive evaluation, convergence detection, and a safety cap. Monotone (insert-only) recursion emits partial progress facts before full convergence and publishes a `complete_through` token once the fixed-point is reached. Non-monotone recursion is rejected at runtime with `RS-1009` and flagged `recursion_dred_required` in `EXPLAIN INCREMENTAL`.

## Changes

### Plan IR
- `PlanNode::Recursion { base, step, max_iterations, monotone }`
- `OpKind::Recursion { max_iterations, monotone }`

### Differentiator
- Monotone recursion → `WeightAdd/v1`, no `not_merge_safe_reason`
- Non-monotone recursion → `WeightAdd/v1` + `RecursionDredRequired` reason

### Physical operator (`rockstream-ops`)
- `RecursiveOp`: semi-naive fixed-point loop with frontier tracking
- Convergence detection: halts when frontier is empty
- Safety cap: `max_iterations` prevents infinite loops
- Monotone mode: rejects negative-weight rows with `RS-1009`
- `tc_step_fn()` helper for transitive closure

### Oracle / tests (`rockstream-oracle`)
- `RecursiveOracle`: batch reference implementation for property testing
- 12 proof tests covering TC DAG, hierarchy, cyclic graph, partial progress,
  plan codec round-trip, diff law assignment, DRed escape hatch, oracle match,
  explain label, and closed-enum coverage

### Supporting changes
- `NotMergeSafeReason::RecursionDredRequired` added to closed enum
- `RS_1009` error code registered
- Catalog codec encode/decode for `PlanNode::Recursion`
- Explain labels in plan and runtime crates
- `op_kinds` / `assert_op_kinds` updated in SQL frontend
- Version bumped `0.21.0` → `0.22.0`
- `sign-offs/v0.22.md` added
