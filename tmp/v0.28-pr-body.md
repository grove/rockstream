## Summary

Implements v0.28.0: Control plane and worker discovery — the Phase 4 entry version.

Adds the control service, worker registration with capacity reporting, in-memory
topology catalog, placement algorithm, mTLS scaffolding, and Tier 1/2 role flows
to the binary.

## What's new

### rockstream-types
- `WorkerId` strong-typed ID added to `ids.rs`
- New `topology` module: `NodeRole`, `CapacityHeadroom`, `WorkerRegistration`, `WorkerInfo`, `ControlMessage`, `WorkerMessage`

### rockstream-control
- `TopologyCatalog`: thread-safe in-memory worker registry with versioned mutations
- `PlacementAlgorithm`: capacity-headroom-aware `choose` and `assign_n` with deterministic tie-breaking
- `ControlService` + `ControlServiceHandle`: TCP server accepting newline-delimited JSON frames, audit-log integration
- `TlsConfig` + `LoadedTls`: mTLS scaffolding, validates PEM files at startup

### rockstream (binary)
- `--role=control` Tier 1: starts `ControlService` on `--control-bind`
- `--role=worker` Tier 1: connects to `--control=<addr>`, registers, runs pipeline
- `--role=all` Tier 2: starts control service and self-registers in one process
- `bootstrap` subcommand: probes a control service and prints readiness
- `--tls-cert / --tls-key / --tls-ca-cert` flags for mTLS scaffolding

## Proof criteria

| Criterion | Evidence |

| Tier 1 start flow works | `run_tier1_control` + `run_tier1_worker`; integration tests in `service.rs` |

| Workers join via `--control=<url>` and appear in topology catalog | `topology_catalog_updated_after_registration` test (3 workers) |
| Topology changes are audited | `worker.registered` and `worker.deregistered` events written to `FileAuditLog` |
| Placement respects `capacity_headroom` | `placement_respects_capacity_headroom` unit test + `assign_n` property test |
| mTLS scaffolding compiles and validates certificates | `tls.rs` unit tests: valid PEM, invalid PEM, missing files |
| Bootstrap command documented in CLI help | `rockstream bootstrap --control <addr>` with `--help` docs |

## Notes

- mTLS scaffolding validates PEM at startup but does not perform TLS handshakes yet. Full handshake integration (tokio-rustls) is deferred to v0.46 per ROADMAP scope.
- TCP protocol uses newline-delimited JSON; binary framing may be added in v0.29/v0.30 if benchmarks show it is needed.
