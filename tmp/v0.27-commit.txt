v0.27.0: Single-shard performance profile

## Summary

Implement v0.27.0: Single-shard performance profile. All ROADMAP exit
criteria are met.

## Changes

### New: `rockstream-types::state_budget`

- `StateBudget` struct with named operator budget, `try_acquire` /
  `release` / `force_acquire`, thread-safe atomic counter, RS-5003 error
  on overflow, `utilization()` fill fraction
- 9 unit tests including concurrent Arc-shared acquisition

### New: `rockstream-ops/benches/ivm_vs_batch.rs`

- Criterion IVM vs batch benchmark at 1% change rate for two operators:
  GROUP BY SUM (100k rows, 1k groups) and Filter (100k rows, 50% pass)
- Sanity tests `ivm_sum_speedup_at_1pct_change_rate` and
  `ivm_filter_speedup_at_1pct_change_rate` assert >=5x speedup (CI-safe
  conservative bound; theoretical >= 100x; precise measurement in
  Criterion HTML report)
- `per_law_rmw_avoidance_ratio_proof`: verifies WeightAdd/v1 and
  SumCount/v1 have 100% RMW avoidance; MaxRegister/v1, MinRegister/v1,
  HyperLogLog/v1, BloomUnion/v1 show 0% (ExtremumRequiresRmw); prints
  full per-law ratio table

### Extended: `rockstream-types::metrics`

- `inc_rmw_avoided` / `inc_rmw_required` per-law RMW tracking
- `rmw_avoidance_ratio(key)` and `rmw_ratio_report()` for diagnostics
- `inc_manifest_write` / `read_manifest_writes` manifest churn counter
- 10 new tests: RMW avoidance ratio proofs for abelian group and
  semilattice laws, manifest counter, full report

### Extended: `rockstream-runtime::epoch_coordinator`

- `commit_epoch` now calls `metrics::inc_manifest_write()` once per
  committed epoch (manifest churn budget proxy)
- New test `manifest_churn_budget_one_write_per_epoch`: verifies each
  `commit_epoch` call increments the manifest counter (proves <= 1
  write/epoch budget gate)

### Extended: `rockstream-ops/benches/storage_budget.rs`

- `run_manifest_churn_gate` function: validates <= 1 manifest write per
  epoch using the global counter
- `manifest_churn_budget_gate` test: runs the gate unconditionally
- Gate wired into `gate_probes` (enabled via `STORAGE_BUDGET_RUN_GATES=1`)

### New: `docs/ivm-operators.md`

v0.27 documentation deliverable: IVM operator reference for every
operator with arrangement layout, merge-law annotation, and EXPLAIN
INCREMENTAL output format. Covers Filter, Project, Map, Aggregate
(SUM/COUNT/AVG), Aggregate (MIN/MAX), Distinct, Set Operations, Inner
Join, Outer Join, Window, Tumble, Top-K, Recursive, Snapshot, ViewRef.
